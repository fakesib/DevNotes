# Kotlin

- Полностью совместим с Java (компилируется в тот же байт-код и работает на той же JVM)
- В одном проекте может быть как Java код, так и Kotlin код и они могут использовать друг друга
- Можно импортировать **любые существующие Java библиотеки**

# Переменные

В Kotlin есть два ключевых слова для объявления переменных:

| Ключевое слово | Изменяемость | Пример | Описание |
| --- | --- | --- | --- |
| `val` | ❌ нельзя изменить | `val name = "Alex"` | Значение задаётся один раз — аналог `final` в Java |
| `var` | ✅ можно изменить | `var age = 25` | Значение можно менять позже |

При создании переменной сначала указывается имя, а потом тип. Если мы объявляем переменную уже со значением, то тип можно не указывать:

**`val constant: String = “Constant”`**

**`var variable = 123`**

# Циклы

Цикл `for` в Kotlin работает **по коллекциям, диапазонам или массивам**.

Он **не использует индекс явно**, как в Java (хотя можно получить его при желании).

```kotlin
// for если нужен индекс
for ((index, fruit) in fruits.withIndex()) {
    println("$index: $fruit")
}

// for
for (i in 1..3) println("for: $i")

// while
var x = 3
while (x > 0) {
    println("while: $x")
    x--
}

// do..while
var y = 0
do {
    println("do..while: $y")
    y++
} while (y < 2)
```

Также можно использовать `break` для выхода из цикла, `continue` пропуская текущую итерация, и метки

## Диапазоны

**Диапазоны - это способ задать последовательность значений** (обычно чисел или символов), с которой потом можно работать в циклах, проверках и т.д.

| Конструкция | Что делает |
| --- | --- |
| `1..5` | от 1 до 5 включительно |
| `1 until 5` | от 1 до 4 (5 не включается) |
| `5 downTo 1` | от 5 до 1 |
| `1..10 step 2` | 1, 3, 5, 7, 9 |

# Функции

Общий синтаксис:

```kotlin
fun имяФункции(параметры): Тип_результата {
    // тело функции
    return значение
}
```

- Если функция **возвращает значение**, указывается его тип.
- Если **ничего не возвращает**, можно указать `Unit` (аналог `void` в Java) — но чаще его **опускают**.

Если функция состоит из **одного выражения**, можно не писать `{}` и `return`:

```kotlin
fun square(x: Int) = x * x
fun greet(name: String) = "Привет, $name!"
```

Можно задавать **значения по умолчанию** для параметров:

```kotlin
fun greet(name: String = "Гость") {
    println("Привет, $name!")
}

greet()          // Привет, Гость!
greet("Алиса")   // Привет, Алиса!
```

Функцию можно объявить **внутри другой функции** — это помогает инкапсулировать логику:

```kotlin
fun outer() {
    fun inner() {
        println("Внутренняя функция")
    }
    inner()
}
```

# NPE

**`NullPointerException` (NPE)** — это ошибка времени выполнения, возникающая, когда программа пытается обратиться к методу или свойству **у объекта, равного `null`**.

Kotlin — **null-safe** язык. Он **разделяет переменные на "nullable" и "non-nullable"**.

### Переменные, которые не могут быть `null`:

```kotlin
var name: String = "Alice"
// name = null ❌ — ошибка компиляции
```

### Переменные, которые могут быть `null`:

Чтобы разрешить `null`, нужно явно добавить `?`:

```kotlin
var name: String? = null
```

---

### Как безопасно работать с nullable-переменными

Kotlin даёт несколько инструментов, чтобы избежать NPE

### Safe call (`?.`)

Выполняет обращение **только если значение не `null`**:

```kotlin
val length = name?.length
```

Если `name == null`, результатом будет `null`, а не ошибка.

---

### Elvis-оператор (`?:`)

Позволяет задать **значение по умолчанию**, если слева `null`:

```kotlin
val length = name?.length ?: 0
```

Если `name == null`, длина будет `0`.

---

### Оператор `!!`

Говорит компилятору: "Я уверен, что здесь не `null`"

Но если вы ошиблись — будет **NPE** 😬

```kotlin
val length = name!!.length  // 💥 если name == null → NPE
```

Использовать `!!` нужно **только если вы на 100% уверены**, что значение не `null`.

# Классы

Шаблон базового класса: 

```kotlin
// Первичный конструктор (автоматически создаёт поля и инициализирует их
class Person(val name: String, var age: Int) {
	var gender: Boolean
	// Инициализационный блок, если нужна доп логика при создании
	init {
		println("Создан новый человек: $name ($age лет)")
	}
	// Вторичный конструктор, обязательно должен вызывать первичный
	constructor(name: String, age: Int, gender: Boolean) : this(name, age) {
     this.gender = false
  }
}
```

### Наследование, интерфейсы и остальное

В Kotlin используется **двоеточие `:`** для наследования и реализации интерфейсов. По умолчанию классы в Kotlin **закрыты для наследования.** Чтобы разрешить наследование — нужно явно указать **`open` .** 

- Kotlin автоматически создаёт **`get/set`**, но можно переопределить их.

```kotlin
// Поддерживает также дефолтные методы 
interface Alive {
	fun mind()
}

open class Animal(val name: String) {
		// Реализация интерфейса
		override fun mind() {
			println("Thinking")
		}
		
    open fun makeSound() {
        println("$name издаёт звук")
    }
}

class Dog(name: String, age: Int) : Animal(name) {
		var age: Int = age
			set(value) { field = if (value >= 0) value else 0 }

    override fun makeSound() {
        println("$name лает 🐶")
    }
}
```

Также есть Data-классы - это класс, предназначенный для хранения данных. Kotlin автоматически создаёт: **`toString()`**, **`equals()`**, **`hashCode()`**, **`copy()`**, **`componentN()`**

```kotlin
data class User(val name: String, val age: Int)

val user1 = User("Аня", 22)
val user2 = user1.copy(age = 23)
println(user2) // User(name=Аня, age=23)
```

Object - синглтоны. Если нужен **единственный экземпляр**:

```kotlin
object Config {
    val version = "1.0"
    fun printInfo() = println("App version: $version")
}

Config.printInfo()
```

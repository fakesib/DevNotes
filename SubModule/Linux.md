# Linux

# SSH подключение

Для подключения к серверу по ssh, необходимо сначала сгенерировать ssh-ключ, приватный и публичный. 

**`ssh-keygen -t ed25519 -C "[your_email@example.com](mailto:your_email@example.com)"`**

Ключи генерируются по пути ~/.ssh

- `~/.ssh/id_ed25519` — **приватный ключ** (никому не передавать)
- `~/.ssh/id_ed25519.pub` — **публичный ключ** (его нужно скопировать на сервер)

Далее добавляем публичный ключ на сервер в файл authorized_keys:

**`echo "вставь_сюда_ключ" >> ~/.ssh/authorized_keys`**

И можно подключаться к серверу командой: 

`ssh user@0.0.0.0`

---

### Jump Host

Также по ssh можно подключаться к компьютерам в закрытой сети через Jump Host. Для использования необходимо добавить публичный ключ на обе машины (jump хост и целевой хост) в `~/.ssh/authorized_keys`

**`ssh -J пользователь@jump-хост пользователь@целевой-хост`**

# TMUX — terminal multiplexer

**tmux** - команда которая позволяет **запускать несколько терминальных сессий** в одном окне, **отключаться от сессии и возвращаться к ней позже** — процессы продолжают работать даже после закрытия SSH-соединения

*Это незаменимо при работе на удалённых серверах, в долгих сборках или когда нужно одновременно смотреть логи, редактировать код и запускать тесты.*

**Основные команды:**

| **Команда** | **Что делает** |
| --- | --- |
| **`tmux`**  | Создаёт новую сессию |
| **`tmux new -s <имя сессии>`** | Создаёт новую сессию с названием |
| **`tmux ls`** | Показать все сессии |
| **`tmux attach -t <имя сессии>`** | Подключаться к существующей сессии |
| **`tmux kill-session -t <имя>`** | Убить существующую сессию |

**Команды внутри tmux сессии:**

Все внутренние команды начинаются с префикса `Ctrl + b` - **Prefix**

| Команда  | Что делает |
| --- | --- |
| **`Prefix + d`** | Выйти из текущей сессии |
| **`Prefix + с`** | Создать новое окно |
| **`Prefix + n` / `p`** | Перейти к следующему (next) или предыдущему (previous) окну |
| **`Prefix + w`**  | Интерактивный список всех окон |
| **`Prefix + %`**  | Разделить окна по вертикали |

Окна в tmux ведут себя как вкладки в браузере.
